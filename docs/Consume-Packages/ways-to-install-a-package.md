## <a name="what-happens-when-a-package-is-installed"></a><span data-ttu-id="368cd-129">O que acontece quando um pacote é instalado</span><span class="sxs-lookup"><span data-stu-id="368cd-129">What happens when a package is installed</span></span>

<span data-ttu-id="368cd-130">Em poucas palavras, as diferentes ferramentas NuGet normalmente criam uma referência a um pacote no arquivo de projeto ou em `packages.config`, depois executam uma restauração de pacote, o que efetivamente instala o pacote.</span><span class="sxs-lookup"><span data-stu-id="368cd-130">Simply said, the different NuGet tools typically create a reference to a package in the project file or `packages.config`, then perform a package restore, which effectively installs the package.</span></span> <span data-ttu-id="368cd-131">A exceção é `nuget install`, que apenas expande o pacote em uma pasta `packages` e não modifica nenhum outro arquivo.</span><span class="sxs-lookup"><span data-stu-id="368cd-131">The exception is `nuget install`, which only expands the package into a `packages` folder and does not modify any other files.</span></span>

<span data-ttu-id="368cd-132">O processo geral é o seguinte:</span><span class="sxs-lookup"><span data-stu-id="368cd-132">The general process is as follows:</span></span>

1. <span data-ttu-id="368cd-133">(Todas as ferramentas, exceto `nuget.exe`) Registre a versão e o identificador de pacote no arquivo de projeto ou em `packages.config`.</span><span class="sxs-lookup"><span data-stu-id="368cd-133">(All tools except `nuget.exe`) Record the package identifer and version into the project file or `packages.config`.</span></span>

1. <span data-ttu-id="368cd-134">Adquira o pacote:</span><span class="sxs-lookup"><span data-stu-id="368cd-134">Acquire the package:</span></span>
    - <span data-ttu-id="368cd-135">Verifique se o pacote (pelo identificador exato ou pelo número da versão) já está instalado na pasta *global-packages* conforme descrito em [Como gerenciar as pastas de pacotes globais e de cache](managing-the-global-packages-and-cache-folders.md).</span><span class="sxs-lookup"><span data-stu-id="368cd-135">Check if the package (by exact identifer and version number) is already installed in the *global-packages* folder as described on [Managing the global packages and cache folders](managing-the-global-packages-and-cache-folders.md).</span></span>

    - <span data-ttu-id="368cd-136">Se o pacote não estiver armazenado na pasta *global-packages*, tente recuperá-lo das fontes listadas nos [arquivos de configuração](Configuring-NuGet-Behavior.md).</span><span class="sxs-lookup"><span data-stu-id="368cd-136">If the package is not in the *global-packages* folder, attempt to retrieve it from the sources listed listed in the [configuration files](Configuring-NuGet-Behavior.md).</span></span> <span data-ttu-id="368cd-137">Para fontes online, primeiro tente recuperar o pacote do cache, a menos que `-NoCache` esteja especificado com comandos `nuget.exe` ou `--no-cache` esteja especificado com `dotnet restore`.</span><span class="sxs-lookup"><span data-stu-id="368cd-137">For online sources, attempt first to retrieve the package from the cache unless `-NoCache` is specified with `nuget.exe` commands or `--no-cache` is specified with `dotnet restore`.</span></span> <span data-ttu-id="368cd-138">(O Visual Studio e o `dotnet add package` sempre usam o cache). Se for usado um pacote do cache, "CACHE" será exibido na saída.</span><span class="sxs-lookup"><span data-stu-id="368cd-138">(Visual Studio and `dotnet add package` always use the cache.) If a package is used from the cache, "CACHE" appears in the output.</span></span> <span data-ttu-id="368cd-139">O cache tem um tempo de expiração de 30 minutos.</span><span class="sxs-lookup"><span data-stu-id="368cd-139">The cache has an expiration time of 30 minutes.</span></span>

    - <span data-ttu-id="368cd-140">Se o pacote não estiver armazenado no cache, tente baixá-lo das fontes listadas na configuração.</span><span class="sxs-lookup"><span data-stu-id="368cd-140">If the package is not in the cache, attempt to download it from the sources listed in the configuration.</span></span> <span data-ttu-id="368cd-141">Se um pacote for baixado, "GET" e "OK" serão exibidos na saída.</span><span class="sxs-lookup"><span data-stu-id="368cd-141">If a package is downloaded, "GET" and "OK" appear in the output.</span></span>

    - <span data-ttu-id="368cd-142">Se o pacote não puder ser adquirido com êxito de nenhuma fonte, a instalação falhará neste ponto com um erro como [NU1103](../reference/errors-and-warnings.md#nu1103).</span><span class="sxs-lookup"><span data-stu-id="368cd-142">If the package cannot be successfully acquired from any sources, installation fails at this point with an error such as [NU1103](../reference/errors-and-warnings.md#nu1103).</span></span> <span data-ttu-id="368cd-143">Os erros de comandos `nuget.exe` mostram apenas a última fonte selecionada, mas implicam que o pacote não estava disponível de nenhuma fonte.</span><span class="sxs-lookup"><span data-stu-id="368cd-143">Note that errors from `nuget.exe` commands show only the last source checked, but implies that the package wasn't available from any source.</span></span>

    <span data-ttu-id="368cd-144">Na aquisição do pacote, a ordem das fontes na configuração do NuGet pode ser aplicável:</span><span class="sxs-lookup"><span data-stu-id="368cd-144">When acquiring the package, the order of sources in the NuGet configuration may apply:</span></span>
      - <span data-ttu-id="368cd-145">No caso dos projetos que usam o formato PackageReference, o NuGet verifica primeiro a pasta local de fontes e os compartilhamentos de rede antes de verificar as fontes HTTP.</span><span class="sxs-lookup"><span data-stu-id="368cd-145">For projects using the PackageReference format, NuGet checks sources local folder and network shares before checking HTTP sources.</span></span>
      - <span data-ttu-id="368cd-146">No caso dos projetos que usam o formato de gerenciamento `packages.config`, o NuGet usa a ordem das fontes na configuração.</span><span class="sxs-lookup"><span data-stu-id="368cd-146">For projects using the `packages.config` management format, NuGet uses the order of the sources in the configuration.</span></span> <span data-ttu-id="368cd-147">Uma exceção são as operações de restauração, caso em que a ordenação das fontes é ignorada e o NuGet usa o pacote da fonte que responder primeiro.</span><span class="sxs-lookup"><span data-stu-id="368cd-147">An exception is restore operations, in which case source ordering is ignored and NuGet uses the package from whichever source responds first.</span></span>
      - <span data-ttu-id="368cd-148">De modo geral, a ordem em que o NuGet verifica as fontes não é especialmente significativa, porque todos os pacotes fornecidos com um identificador e número de versão específicos são exatamente os mesmos em todas as fontes encontradas.</span><span class="sxs-lookup"><span data-stu-id="368cd-148">In general, the order in which NuGet checks sources isn't particularly meaningful, because any given package with a specific identifier and version number is exactly the same on whatever source it's found.</span></span>

1. <span data-ttu-id="368cd-149">(Todas as ferramentas, exceto `nuget.exe`) salvam uma cópia do pacote e outras informações na pasta *http-cache*, conforme descrito em [Como gerenciar as pastas de pacotes globais e de cache](managing-the-global-packages-and-cache-folders.md).</span><span class="sxs-lookup"><span data-stu-id="368cd-149">(All tools except `nuget.exe`) Save a copy of the package and other information in the *http-cache* folder as described on [Managing the global packages and cache folders](managing-the-global-packages-and-cache-folders.md).</span></span>

1. <span data-ttu-id="368cd-150">Se o pacote for baixado, instale-o na pasta *global-packages* por usuário.</span><span class="sxs-lookup"><span data-stu-id="368cd-150">If downloaded, install the package into the per-user *global-packages* folder.</span></span> <span data-ttu-id="368cd-151">O NuGet cria uma subpasta para cada identificador de pacote e, em seguida, cria subpastas para cada versão instalada do pacote.</span><span class="sxs-lookup"><span data-stu-id="368cd-151">NuGet creates a subfolder for each package identifier, then creates subfolders for each installed version of the package.</span></span>

1. <span data-ttu-id="368cd-152">Atualize outros arquivos de projeto e pastas:</span><span class="sxs-lookup"><span data-stu-id="368cd-152">Update other project files and folders:</span></span>

    - <span data-ttu-id="368cd-153">Em caso de projetos que usam PackageReference, atualize o grafo de dependência do pacote armazenado em `obj/project.assets.json`.</span><span class="sxs-lookup"><span data-stu-id="368cd-153">For projects using PackageReference, update the package dependency graph stored in `obj/project.assets.json`.</span></span> <span data-ttu-id="368cd-154">O conteúdo do pacote em si não é copiado para nenhuma pasta de projeto.</span><span class="sxs-lookup"><span data-stu-id="368cd-154">Package contents themselves are not copied into any project folder.</span></span>
    - <span data-ttu-id="368cd-155">Em projetos que usam `packages.config`, copie as partes do pacote expandido que correspondem à estrutura de destino do projeto na pasta `packages` do projeto.</span><span class="sxs-lookup"><span data-stu-id="368cd-155">For projects using `packages.config`, copy those parts of the expanded package that match the project's target framework into project's `packages` folder.</span></span> <span data-ttu-id="368cd-156">(Quando `nuget install` é usado, todo o pacote expandido é copiado, pois `nuget.exe` não examina arquivos de projeto para identificar a estrutura de destino).</span><span class="sxs-lookup"><span data-stu-id="368cd-156">(When using `nuget install`, the entire expanded package is copied because `nuget.exe` does not examine project files to identify the target framework.)</span></span>
    - <span data-ttu-id="368cd-157">Atualize `app.config` e/ou `web.config` se o pacote usar [transformações de origem e arquivo de configuração](../create-packages/source-and-config-file-transformations.md).</span><span class="sxs-lookup"><span data-stu-id="368cd-157">Update `app.config` and/or `web.config` if the package uses [source and config file transformations](../create-packages/source-and-config-file-transformations.md).</span></span>

1. <span data-ttu-id="368cd-158">Instale todas as dependências de nível inferior se não estiverem presentes no projeto.</span><span class="sxs-lookup"><span data-stu-id="368cd-158">Install any down-level dependencies if not already present in the project.</span></span> <span data-ttu-id="368cd-159">Esse processo pode atualizar versões do pacote, conforme descrito em [Resolução de dependências](../consume-packages/dependency-resolution.md).</span><span class="sxs-lookup"><span data-stu-id="368cd-159">This process might update package versions in the process, as described in [Dependency Resolution](../consume-packages/dependency-resolution.md).</span></span>

1. <span data-ttu-id="368cd-160">(Somente no Visual Studio) Exiba o arquivo Leiame do pacote, se disponível, em uma janela do Visual Studio.</span><span class="sxs-lookup"><span data-stu-id="368cd-160">(Visual Studio only) Display the package's readme file, if available, in a Visual Studio window.</span></span>
