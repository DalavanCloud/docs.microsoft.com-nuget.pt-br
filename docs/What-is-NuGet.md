<span data-ttu-id="9deb9-175">Como projetos podem ser movidos facilmente entre os computadores de desenvolvedor, repositórios de controle do código-fonte, servidores de build e assim por diante, é altamente impraticável manter assemblies binários dos pacotes do NuGet diretamente associados a um projeto.</span><span class="sxs-lookup"><span data-stu-id="9deb9-175">Because projects can easily move between developer computers, source control repositories, build servers, and so forth, it's highly impractical to keep binary assemblies from NuGet packages directly bound to a project.</span></span> <span data-ttu-id="9deb9-176">Isso não apenas tornaria cada cópia do projeto desnecessariamente sobrecarregada (desperdiçando, portanto, espaço em repositórios de controle do código-fonte), isso também dificultaria muito a atualização de binários do pacote para versões mais recentes, pois isso precisaria ser feito em todas as cópias do projeto.</span><span class="sxs-lookup"><span data-stu-id="9deb9-176">Not only would this make each copy of the project unnecessarily bloated (and thereby waste space in source control repositories), it would also make it very difficult to update package binaries to newer versions as this would have to be done across all copies of the project.</span></span> 

<span data-ttu-id="9deb9-177">Em vez disso, o NuGet simplesmente mantém uma lista de referência de pacotes dos quais um projeto depende (incluindo as dependências de nível superior e inferior) e fornece os meios para restaurar todos os pacotes referenciados mediante solicitação, conforme descrito em [Restauração de pacote](Consume-Packages/Package-Restore.md).</span><span class="sxs-lookup"><span data-stu-id="9deb9-177">Instead, NuGet simply maintains a reference list of the packages upon which a project depends (including both top-level and down-level dependencies), and provides the means to restore all referenced packages upon request as described on [Package restore](Consume-Packages/Package-Restore.md).</span></span> <span data-ttu-id="9deb9-178">Ou seja, sempre que você instala um pacote de algum host em um projeto, o NuGet registra o identificador de pacote e o número de versão nesta lista de referência.</span><span class="sxs-lookup"><span data-stu-id="9deb9-178">That is, whenever you install a package from some host into a project, NuGet records the package identifier and version number in this reference list.</span></span> <span data-ttu-id="9deb9-179">(Desinstalar um pacote, é claro, remove-o da lista.)</span><span class="sxs-lookup"><span data-stu-id="9deb9-179">(Uninstalling a package, of course, removes it from the list.)</span></span>
